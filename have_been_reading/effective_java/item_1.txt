Consider static factory methods instead of constructors

- When thinking of providing ways to instantiate a class:
    - we know public constructors 
    - another way = a static method that returns an instance of the class
    - Example: Boolean class (the boxed primitive class for boolean
    - A class can provide its clients with static factory methods instead of, or in addition to, public constructors.
    
- Adv of static factory methods
    - Unlike constructors, they have names.
        - code easier to read
        - In cases where a class seems to require multiple constructors with the same signature, replace the constructors with static factory methods and carefully chosen names to highlight their differences.
    - They are not required to create a new object each time they’re invoked
        - This allows immutable classes (Item 17) to use preconstructed instances, or to cache instances as they’re constructed, and dispense them repeatedly to avoid creating unnecessary duplicate objects.
        - Example:
            public final class Boolean {
                // Pre-constructed instances (immutable and safe to share)
                public static final Boolean TRUE = new Boolean(true);
                public static final Boolean FALSE = new Boolean(false);
                
                private final boolean value;
                
                // Private constructor
                private Boolean(boolean value) {
                    this.value = value;
                }
                
                // Static factory - returns cached instances
                public static Boolean valueOf(boolean b) {
                    return b ? TRUE : FALSE;  // Reuses same objects!
                }
            }

            // Usage
            Boolean b1 = Boolean.valueOf(true);
            Boolean b2 = Boolean.valueOf(true);
            Boolean b3 = Boolean.valueOf(true);

            System.out.println(b1 == b2);  // true (same object!)
            System.out.println(b2 == b3);  // true (same object!)

            // Only TWO Boolean objects exist in entire JVM
            // Safe because Boolean is immutable
